# Submission checklist

Before public submission, the author or publishing agent should complete every
item.

- [ ] Confirm author name, affiliation, email, and ORCID.
- [ ] Run a fresh Lean build from an empty Lake cache.
- [ ] Record the CI URL and final commit hash in the paper.
- [ ] Confirm the upstream commit remains publicly accessible.
- [ ] Run a fresh literature search for overlapping formalizations.
- [ ] Decide whether to submit as a formalization paper, artifact paper, or
      mathematical-physics note.
- [ ] Confirm journal/conference page limit and bibliography style.
- [ ] Replace the provisional licensing note with the publisher-approved one.
- [ ] Proofread every statement that uses "first", "new", or "formalized".
- [ ] Confirm the limitations paragraph remains in the abstract and conclusion.
- [ ] Upload the exact ZIP used for artifact evaluation to Zenodo or an
      institutional archive and add the DOI to `CITATION.cff`.
- [ ] Compile the final PDF twice and inspect every page visually.
